# MVP implementation plan

1. **Safe foundation (implemented here):** exact host scope, passive portal/application detection, replacement-aware iframe registry, responsive shell, error/session states, original-UI escape hatch, synthetic fixtures, network-denied tests, and an allowlisted academic-process catalog that visibly blocks consequential routes.
2. **Responsive sign-in (implemented):** place the existing native credential fields and native submit control inside a responsive presentation while preserving the original form, autofill, event handlers, and an immediate original-UI escape hatch. Credential values never enter extension code.
3. **First functional read (implemented):** navigate to either observed grade-query entry through exact allowlisted menu labels after an explicit user action; share the fingerprinted grade application; select a visible program; optionally include withdrawn courses; select one academic period; and display only that period's normalized course rows responsively in memory.
4. **Academic-offer search (implemented):** navigate through the exact `Matrícula Pregrado > Oferta Académica` labels, delegate an explicit course-code search or Sirius's visible name-or-code value help, and normalize selector results plus the verified 12-column response into responsive rows. Prefix searches such as `FGE` remain user-triggered; selecting an exact result performs one offer search. Initial, results, empty, loading, invalid, stale-control, and mobile states use synthetic fixtures or tests. No search-on-type or automatic retry.
5. **Next safe milestone:** add fixture-driven parsing for already-rendered period statistics, then study current enrollment and schedule. Registration-window values remain outside automated discovery.
6. **Local planning:** build a registration planner that never contacts SAP.
7. **Transactional work:** remains out of scope until a separate, explicitly authorized design and validation effort establishes confirmation boundaries and complete sanitized state-machine fixtures.
